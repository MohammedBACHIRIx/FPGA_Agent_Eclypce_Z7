# 组合模块搭建说明（以“正弦波发生器”为参考）

## 1. 目标

- `qt_module.py` 中定义组合模块类（继承 `CompositeModule`）。
- `qt_ui_graph.py` 中把组合模块加入 `DiagramView.composite_modules`，并在 `ComponentPalette` 里展示。

---

## 2. 当前机制（你项目里的真实实现）

### 2.1 组合模块基类

文件：`qt_module.py`

- `class CompositeModule`
	- 关键函数：`create_sub_modules(cls, scene, position, alloc_index_func)`
	- 行为：遍历 `sub_modules`，通过 `module_factory` 找到子模块类，分配索引并实例化加入场景。
	- 优点：自动复用现有模块的数量上限、参数体系、连线规则、模式切换（free/developer）。

### 2.2 已有组合模块范例

文件：`qt_module.py`

- `class SINGenerator(CompositeModule)`
- `class DigitalControlledOscillator(CompositeModule)`
- `composite_modules = {"正弦波发生器": SINGenerator, "数字控制振荡器": DigitalControlledOscillator, ...}`

### 2.3 画布拖拽创建入口

文件：`qt_ui_graph.py`

- `class DiagramView`
	- `composite_modules = {...}`
	- `dropEvent(self, event)`：
		- 若拖拽文本命中组合模块名，则调用 `composite_cls.create_sub_modules(...)`。
		- 否则按普通基础模块创建。

### 2.4 模块列表显示入口

文件：`qt_ui_graph.py`

- `class ComponentPalette`
	- `composite_items = ["正弦波发生器", "数字控制振荡器", ...]`
	- 组合模块显示在“组合模块”分组。

## 3. 新增一个组合模块的最小改动清单

只做以下 3 处改动即可，不会影响已有模块逻辑：

1. `qt_module.py`
2. `qt_ui_graph.py`
3. （可选）`模块搭建说明.md` 记录你新增模块说明

### 3.1 在 `qt_module.py` 增加组合类 + 注册

在 `CompositeModule` 相关位置新增：

```python
class MyComposite(CompositeModule):
		sub_modules = [
				("累加器", QPointF(0, 0)),
				("三角函数运算器", QPointF(220, 0)),
				# 继续加子模块（名称必须存在于 module_factory 键中）
		]
```

然后把它加入 `composite_modules`：

```python
composite_modules = {
		"正弦波发生器": SINGenerator,
		"数字控制振荡器": DigitalControlledOscillator,
		"我的组合模块": MyComposite,
}
```

注意：

- `sub_modules` 里的模块名必须和 `module_factory` 的键完全一致。
- 偏移量 `QPointF(dx, dy)` 是相对拖拽落点的坐标，建议先粗排，后微调。

### 3.2 在 `qt_ui_graph.py` 导入并注册

1) 顶部 import 增加新组合类：

```python
from qt_module import (
		...,
		MyComposite,
		...
)
```

2) `DiagramView.composite_modules` 增加映射：

```python
composite_modules = {
		"正弦波发生器": SINGenerator,
		"数字控制振荡器": DigitalControlledOscillator,
		"我的组合模块": MyComposite,
}
```

### 3.3 在 `ComponentPalette` 增加显示项

文件：`qt_ui_graph.py`

`ComponentPalette.__init__` 中：

```python
composite_items = ["正弦波发生器", "数字控制振荡器", "我的组合模块"]
```

到这里就可以在 UI 中拖拽创建你的组合模块了。

## 4. 示例：新增“锁相正弦源（示例）”

> 只是示例名称，你可以改成自己的业务名。

在 `qt_module.py`：

```python
class LockedSinSource(CompositeModule):
		sub_modules = [
				("累加器", QPointF(0, 0)),
				("三角函数运算器", QPointF(220, -80)),
				("线性缩放器", QPointF(440, -80)),
		]

composite_modules = {
		"正弦波发生器": SINGenerator,
		"数字控制振荡器": DigitalControlledOscillator,
		"锁相正弦源": LockedSinSource,
}
```

在 `qt_ui_graph.py`：

```python
from qt_module import (..., LockedSinSource, ...)

class DiagramView(QGraphicsView):
		composite_modules = {
				"正弦波发生器": SINGenerator,
				"数字控制振荡器": DigitalControlledOscillator,
				"锁相正弦源": LockedSinSource,
		}

class ComponentPalette(QListWidget):
		def __init__(self):
				...
				composite_items = ["正弦波发生器", "数字控制振荡器", "锁相正弦源"]
```

## 5. 为什么不会影响原来模块使用

你的当前架构是“增量式扩展”，不是“替换式扩展”：

- 组合模块只是新增一个拖拽入口，本质上仍调用已有基础模块构造逻辑。
- 原有 `module_factory`、已有模块类、参数窗口、连线逻辑都不需要改动。
- `DiagramView._alloc_index` 会按原有数量上限分配索引，防止冲突。
- 数量上限超出时只会打印错误，不会破坏已有节点。

因此只要你不改已有模块类的内部行为，新增组合模块就是“旁路增加功能”。

## 6. 何时需要改更多文件（进阶场景）

如果你的组合模块只由“已有基础模块”拼接，不需要额外修改。  
但如果你还要引入“全新的基础模块类型”，还需要：

1. `qt_module.py`
- 新增模块类（继承 `NodeItem`）
- 补充 `module_factory`
- 补充 `module_maxm`

2. `qt_ui_graph.py`
- `DiagramView.module_factory` 补充该模块
- `ComponentPalette.normal_items` 补充显示

3. `qt_ui_utils.py` / `port_numbers.py`（仅当该模块要参与硬件端口映射）
- 新增命名解析和端口号映射

4. `qt_ui_mainwindow.py`（仅当该模块需要特殊业务逻辑）
- 检查 `run_business_logic`
- 检查 `handle_connection_removed`

## 7. 推荐验证流程（每次新增组合模块后都跑一遍）

1. 启动 UI，确认新组合模块出现在“组合模块”分组。  
2. 拖拽一次新组合模块，检查所有子模块都按预期位置生成。  
3. 检查每个子模块索引是否正常递增、名称是否不冲突。  
4. 连线并观察底部日志，确认连接建立/断开行为正常。  
5. 保存配置后重新加载，确认节点与连线恢复正确。  
6. 再单独拖拽原有基础模块，确认原流程不受影响。

## 8. 常见错误与排查

1. 组合模块拖拽后没反应  
原因：`qt_ui_graph.py` 里没加入 `DiagramView.composite_modules`。  
排查：检查 `dropEvent` 是否命中组合名。

2. 只生成部分子模块  
原因：子模块名写错，`module_factory.get(sub_name)` 返回 `None`；或超出数量上限。  
排查：看日志是否有“超出模块数量上限”。

3. 画布上节点重叠  
原因：`sub_modules` 偏移过近。  
排查：增大 `QPointF(dx, dy)` 间距，先横向 200~260 像素为起点。

4. 模块列表看不到新组合模块  
原因：忘记加入 `ComponentPalette` 的 `composite_items`。

## 9. 你当前代码中最关键的函数入口（速查）

- `qt_module.py`
	- `CompositeModule.create_sub_modules`

- `qt_ui_graph.py`
	- `DiagramView.dropEvent`
	- `DiagramView._alloc_index`
	- `DiagramView.composite_modules`
	- `ComponentPalette.__init__`（`composite_items`）

- `qt_ui_mainwindow.py`（通常无需改，仅理解链路）
	- `run_business_logic`
	- `handle_connection_removed`

## 10. 结论

- 在 `qt_module.py` 定义组合类并注册；
- 在 `qt_ui_graph.py` 注册拖拽识别并加入模块列表。
