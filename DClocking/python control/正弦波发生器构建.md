# 正弦波发生器构建说明

## 1. 目的与结果

本文档说明如何在当前项目中，搭建一个“正弦波发生器”组合模块。目标是：

1. 在模块列表中出现“正弦波发生器”。
2. 将其拖到画布后，一次性创建多个基础模块（例如“累加器 + 三角函数运算器”）。
3. 不影响原有基础模块与其他组合模块的使用。

在你当前代码中，这个功能已经存在（`SINGenerator`），本文重点是讲清楚完整流程，便于你后续复制同样方法扩展新组合模块。

---

## 2. 架构总览（先理解再动手）

组合模块的创建链路如下：

1. `qt_module.py`
定义组合模块类（继承 `CompositeModule`），并配置子模块清单 `sub_modules`。

2. `qt_ui_graph.py`
在 `DiagramView.composite_modules` 中注册组合模块名称到类的映射。

3. `qt_ui_graph.py`
在 `ComponentPalette` 的 `composite_items` 中加入显示名称。

4. 拖拽触发
`DiagramView.dropEvent` 判断拖入名称是否属于组合模块，若是则调用 `create_sub_modules(...)` 批量创建。

这个方案属于“新增入口 + 复用基础模块构造”，因此天然低风险。

---

## 3. 正弦波发生器的原理映射（业务层）

一个最小正弦波发生器通常可表示为：

1. 累加器：产生相位累加值。
2. 三角函数运算器：将相位映射为 `SIN/COS`。

所以对应子模块组合可写成：

1. `("累加器", QPointF(0, 0))`
2. `("三角函数运算器", QPointF(200, 0))`

这正是当前 `SINGenerator` 的实现。

---

## 4. 关键代码位置与职责

### 4.1 `qt_module.py`

#### A. 组合模块基类

`CompositeModule.create_sub_modules(cls, scene, position, alloc_index_func)` 职责：

1. 遍历 `cls.sub_modules`。
2. 从 `module_factory` 找到子模块构造类。
3. 通过 `alloc_index_func` 分配实例索引（受数量上限控制）。
4. 在 `scene` 中创建并加入节点。
5. 继承当前模式（free/developer）的参数编辑行为。

#### B. 正弦波发生器定义

```python
class SINGenerator(CompositeModule):
	sub_modules = [
		("累加器", QPointF(0, 0)),
		("三角函数运算器", QPointF(200, 0)),
	]
```

#### C. 注册到组合模块字典

```python
composite_modules = {
	"正弦波发生器": SINGenerator,
	"数字控制振荡器": DigitalControlledOscillator,
}
```

说明：

1. `sub_modules` 中的名称必须与 `module_factory` 键完全一致。
2. 偏移量是相对落点坐标，单位为画布像素。

---

### 4.2 `qt_ui_graph.py`

#### A. `DiagramView.composite_modules`

这里是 UI 侧的组合模块注册表，拖拽时用于匹配。

```python
composite_modules = {
	"正弦波发生器": SINGenerator,
	"数字控制振荡器": DigitalControlledOscillator,
}
```

#### B. `DiagramView.dropEvent(self, event)`

核心逻辑：

1. `component_name = event.mimeData().text()` 取拖拽名字。
2. 若 `component_name in self.composite_modules`，调用：

```python
composite_cls.create_sub_modules(self.scene(), position, self._alloc_index)
```

3. 否则走普通基础模块创建。

#### C. `ComponentPalette` 展示

`composite_items` 中的内容决定组合模块是否出现在列表里：

```python
composite_items = ["正弦波发生器", "数字控制振荡器"]
```

## 5. 从零新建一个“正弦波发生器类组合模块”的步骤

以下步骤适用于你后续新增同类组合模块（例如“正弦波发生器2”）。

### 步骤 1：确认基础模块可用

检查 `qt_module.py` 的 `module_factory` 是否包含：

1. `"累加器"`
2. `"三角函数运算器"`

如果基础模块本身不在 `module_factory`，组合模块无法创建对应子模块。

### 步骤 2：定义组合模块类

在 `qt_module.py` 中添加类：

```python
class MySinGenerator(CompositeModule):
	sub_modules = [
		("累加器", QPointF(0, 0)),
		("三角函数运算器", QPointF(220, 0)),
	]
```

建议：先用 180~260 像素水平间距，避免节点重叠。

### 步骤 3：注册到 `qt_module.py` 的 `composite_modules`

```python
composite_modules = {
	"正弦波发生器": SINGenerator,
	"数字控制振荡器": DigitalControlledOscillator,
	"正弦波发生器2": MySinGenerator,
}
```

### 步骤 4：导入并注册到 `qt_ui_graph.py`

1. `from qt_module import ... MySinGenerator ...`
2. `DiagramView.composite_modules` 加上：

```python
"正弦波发生器2": MySinGenerator
```

### 步骤 5：加入左侧模块列表

在 `ComponentPalette.__init__` 的 `composite_items` 增加：

```python
composite_items = ["正弦波发生器", "数字控制振荡器", "正弦波发生器2"]
```

### 步骤 6：运行验证

运行入口：`qt_UI1.py`

检查项：

1. 左侧“组合模块”分组能看到新条目。
2. 拖拽后一次生成两个子模块。
3. 名称与编号合理，不与现有模块冲突。
4. 原有模块拖拽、参数、连线仍然正常。

---

## 6. 为什么不会影响原有模块

关键原因：

1. 组合模块不是替换基础模块，而是复用已有模块构造流程。
2. `dropEvent` 只在拖入组合模块名称时触发组合创建分支。
3. 原有模块依然走原始分支，不改变原行为。
4. 索引分配由 `_alloc_index` 统一管理，避免同类实例冲突。

只要你不改基础模块类的内部逻辑，新增组合模块通常不会引入回归。

---

## 7. 当前实现的能力边界（很重要）

当前 `CompositeModule.create_sub_modules` 只做“批量生成节点”，不自动连线。也就是说：

1. 拖入“正弦波发生器”后会生成“累加器”和“三角函数运算器”。
2. 两者之间的连接仍需要你手动拉线。

---

## 8. 扩展：自动连线版本

可在 `CompositeModule` 增加一个“自动连线描述”，例如：

```python
class MySinGenerator(CompositeModule):
	sub_modules = [
		("累加器", QPointF(0, 0)),
		("三角函数运算器", QPointF(220, 0)),
	]
	auto_edges = [
		(0, 0, 1, 0),  # 第0个模块 Out1 -> 第1个模块 In1
	]
```

然后在 `create_sub_modules` 中：

1. 先把创建出的节点按顺序存入列表 `nodes`。
2. 遍历 `auto_edges`，取 `start_port` 与 `end_port`。
3. 创建 `EdgeItem(start_port, end_port)` 并加入场景。
4. 同步维护 `node.edges` 列表与必要信号。

注意：

1. 需要复用现有连线合法性规则（端口类型、信号类型、目标输入占用）。
2. 自动连线实现不当容易引入重复边或状态不一致。
3. 建议先跑手动连线稳定版，再做自动连线增强。

---

## 9. 常见问题排查

### 问题 1：拖拽后没有生成任何模块

排查：

1. 是否注册到 `DiagramView.composite_modules`。
2. 拖拽文本名称是否与字典键完全一致。
3. 是否正确导入对应组合类。

### 问题 2：只生成了一部分子模块

排查：

1. `sub_modules` 中是否有拼写错误，导致 `module_factory.get(sub_name)` 为空。
2. 是否触发“超出模块数量上限”。

### 问题 3：新组合模块不出现在左侧列表

排查：

1. 是否加入 `ComponentPalette` 的 `composite_items`。

### 问题 4：新模块影响旧模块

排查：

1. 是否误改了基础模块类实现。
2. 是否修改了普通模块创建分支代码。

---

## 10. 实践建议

建议按以下节奏开发，风险最小：

1. 先只加组合类和注册，不加自动连线。
2. 先验证“能拖出子模块 + 不影响旧功能”。
3. 再考虑自动连线和更复杂布局。
4. 每次改动后做一次“拖拽/连线/保存/加载”回归。

---

## 11. 一页速记（最小必做）

仅需做 3 件事：

1. 在 `qt_module.py` 新增组合类并加入 `composite_modules`。
2. 在 `qt_ui_graph.py` 的 `DiagramView.composite_modules` 注册。
3. 在 `qt_ui_graph.py` 的 `ComponentPalette.composite_items` 显示。

完成后即可得到一个和“正弦波发生器”同机制的组合模块。
