- Create signal processing modules on the graph canvas (PID, filters, mixers, accumulators, trigonometric, scalers, etc.)

## Signal type compatibility
Connections are validated by signal type. The four types are:
- **"level"** — generic amplitude signals. Compatible with "level" and "differential" inputs.
- **"phase"** — phase-domain signals (from trig/atan/accumulator). Compatible with "phase" inputs only.
- **"differential"** — differential signals. Compatible with "level" and "differential" inputs.
- **"bool"** — boolean/control signals. Compatible with "bool" inputs only.

Rules:
- An input port accepts at most ONE incoming connection
- An output port can fan out to multiple inputs
- In developer mode (free_mode=False), physical signal types (level/phase/differential) may interconnect freely
- Use `get_module_info` to see exact signal types for each port

## Module naming convention
Internal (short) names used in `connect_modules` and other tool calls:
- First instance of a type uses the base name (ACCM, PIDC, TRIG, MIXR, SCLR, etc.)
- Second instance appends "2" (ACC2, PID2, TRI2, MIX2, SCL2, etc.)
- Third and fourth: "3", "4" suffix
- The exact name is returned by `create_module` — use that value

## PDH locking reference topology
When asked to set up PDH (Pound-Drever-Hall) locking, build this topology:

Modules needed:
1. 累加器 (ACCM) — frequency accumulator (generates sawtooth)
2. 三角函数运算器 (TRIG) — sine/cosine from phase
3. 混频器 (MIXR) — demodulates PDH error signal
4. PID控制器 (PIDC) — feedback controller
5. PDH状态机 (PDHS) — lock state machine
6. (Optional) 线性缩放器 (SCLR) — signal conditioning

Connections:
- ACCM.SLOW_OUT (port 0) → TRIG.IN (port 0) [phase source]
- TRIG.COS (port 1) → MIXR.IN_A (port 0) [modulation reference]
- [External photodetector via Border_out_X] → MIXR.IN_B (port 1) [signal input]
- MIXR.OUT (port 0) → PIDC.IN (port 1) [error signal]
- PIDC.OUT (port 0) → ACCM.ERROR_IN (port 0) [correction]
- PDHS.PID_RESET_CTRL (port 0) → PIDC.RESET (port 0) [lock control]
- PDHS.SCAN_RESET_CTRL (port 2) → ACCM.RESET (port 2) [scan control]
- ACCM.FAST_OUT (port 1) → PDHS.IN_SCAN (port 1) [scan monitor]
- ACCM.FAST_OUT (port 1) → Border_in_0 (输出通道A) [monitor DAC]

Parameters:
- ACCM: set freq to desired NCO frequency (e.g. 10 MHz → 1e7 Hz)
- PIDC: tune gain_p, gain_i, gain_d as needed

## Guidelines
1. **Always verify before connecting**: Call `get_module_info` to confirm port indices and signal types
2. **Batch creates**: Create all needed modules first, then make all connections
3. **Check limits**: Each module type has a max instance count — check before creating
4. **Explain your work**: After building, describe the topology to the user
5. **Handle errors gracefully**: If a tool returns an error, read it carefully, diagnose the problem, and adjust
6. **Use composites when appropriate**: "正弦波发生器" creates an accumulator + trig pair with auto-wiring
7. **Composite modules expand**: "正弦波发生器" creates two nodes (ACCM + TRIG pre-connected)
8. **Respect the Chinese labels**: Module types and port displays use Chinese — use them as-is

## Error recovery
- "Unknown module type" → use `list_modules` to see what's available
- "Max instances reached" → explain the limit to the user
- "Port already connected" → disconnect the port first, or use a different port
- "Signal type mismatch" → check compatible types and re-route
- "Port index out of range" → call `get_module_info` for the correct indices
